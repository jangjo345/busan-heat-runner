# 부산 러너 🏃‍♂️🔥

부산의 한여름 폭염 시즌을 살아남는 **러너 펫 생존 게임**.
다마고치 + 텍스트 어드벤처 하이브리드. **웹(브라우저)** 버전과 **CLI** 버전 둘 다 있다.

## ▶ 웹 버전 (추천 — 그냥 더블클릭)

파일 탐색기에서 **`index.html` 더블클릭** → 크롬에서 바로 실행. 설치도, cmd도 필요 없다.
(말랑 파스텔 비주얼, 시간대별 하늘·날씨 연출, 클릭 선택지, 진행 상황은 브라우저에 자동 저장)

> 모든 게임 로직·데이터가 `index.html` 한 파일에 내장돼 있어 인터넷/서버 없이 돈다.
> 로컬 서버로 띄우고 싶으면 `npm run web` → http://localhost:4321

## ▶ CLI 버전 (터미널)

```bash
npm install
npm start          # 이어하기(세이브 있으면) 또는 새 게임
npm run new        # 세이브 무시하고 새 게임 시작
```

## 규칙

- 목표: 러너 펫이 부산 여름 **7일차**를 무사히 완주.
- 시간은 실시간이 아니라 **일차 + 시간대**(새벽·오전·정오·오후·저녁·밤)로만 흐른다. 선택할 때마다 한 칸씩 진행.
- 매번 `saves/save.json`에 저장되어, 다시 켜면 이어서 한다.

### 스탯 (0~100, 체온만 예외)

| 스탯 | 설명 |
|------|------|
| 수분 | 0이 되면 **탈수 리타이어**. 물을 마시면 회복 |
| 체온 | **36~38이 안전**. 40 이상이면 **열사병 리타이어**. 그늘·물·휴식으로 낮춘다 |
| 체력 | 달리면 소모, 쉬면 회복 |
| 사기 | 0이 되면 **포기 엔딩**. 좋은 선택·완주로 상승 |

- **정오/오후**: 체온 급상승·수분 급감 (가장 위험)
- **새벽/저녁/밤**: 회복 구간, 러닝하기 좋음

> **공정성 규칙:** 시간대/장비 패시브만으로는 체온이 39까지만 오르고, 수분·사기도 1 미만으로 떨어지지 않는다.
> 즉 리타이어(체온 40↑·수분 0·사기 0)는 **항상 네 선택의 결과**다 — 손쓸 기회 없이 죽지 않는다.

## 장비 시스템 (SUMMERTECT)

맨몸으로 출발하고, 진행 중 **"보급" 이벤트**에서 장비를 얻어 장착하면 패시브 효과가 붙는다.
효과는 실제 제품 스펙에서 그대로 유도한다("기능=수치").

| 장비 | 슬롯 | 스펙 | 효과 |
|------|------|------|------|
| Flexer 캡 | 머리 | UPF 차단·47g·packable | 정오·오후 체온 상승 -30% |
| 여름 헤어밴드 | 머리 (캡과 택일) | 땀 흡수 밴드 | 수분 소모 -15% |
| Frosty 넥 게이터 | 목 | 흡습·냉감 원단 | 턴마다 체온 -1 |
| 카프 슬리브 | 다리 | 단계별 압박 | 달릴 때 체력 소모 -20% |
| 듀얼레이어 런쇼츠 | 하의 | 2겹·무마찰 | 턴마다 사기 +1 |

- 슬롯: 머리 / 목 / 다리 / 하의. 머리 슬롯은 **캡 ↔ 헤어밴드 택일**(나중 장착이 교체).
- 수치는 `data/items.json`에서만 조정한다(엔진은 mod의 `kind`만 해석).

## 구조 (데이터 주도)

```
index.html            # ★ 웹 버전 (단일 파일, 더블클릭 실행 — 데이터 내장)
serve.js              # 웹 버전 로컬 서버 (npm run web)
index.js              # CLI 엔트리
src/
  game.js             # 게임 상태 + 메인 루프
  pet.js              # 펫 스탯/상태 계산 (+ 패시브 공정성 규칙)
  time.js             # 일차·시간대 진행 + 시간대별 패시브 효과
  events.js           # 이벤트 로드·선택 로직 (보급/특수/일반 우선순위)
  items.js            # 장비 시스템 엔진 (mod 해석·장착·스케일링)
  save.js             # JSON 저장/불러오기
  ui.js               # 콘솔 출력, ASCII 펫 얼굴, 컬러
data/
  events.json         # 이벤트 카드(상황문 + 선택지 + 스탯 효과 + 등장 조건)
  items.json          # 장비 효과 수치 (여기 숫자만 바꿔 튜닝)
  endings.json        # 엔딩 분기
saves/                # save.json (gitignore)
```

### 이벤트 등장 우선순위

같은 시간대/일차에 여러 카드가 매칭되면: **① 미소비 보급 이벤트 → ② 일차 지정 특수 이벤트 → ③ 일반 이벤트** 순으로 우선한다.
덕분에 보급/특수 이벤트는 지정한 날에 확실히 등장하고, 일반 카드는 나머지 칸을 채운다.

### 이벤트 카드 추가하는 법

`data/events.json`에 카드 객체만 추가하면 게임이 늘어난다.

```json
{
  "id": "고유_id",
  "title": "카드 제목",
  "text": "상황 설명문 (\\n 으로 줄바꿈)",
  "when": { "bands": ["정오", "오후"], "days": [3, 4] },
  "choices": [
    {
      "label": "선택지 문구",
      "effects": { "체온": -6, "수분": 4, "사기": -1 },
      "result": "선택 결과 텍스트"
    }
  ]
}
```

- `when.bands` / `when.days`: 비우면(`[]`) **아무 때나** 등장. 값이 있으면 해당 시간대/일차에만.
- `effects`: `수분 / 체온 / 체력 / 사기` 중 변화시킬 것만 적는다(+/-).

### 보급(장비 획득) 이벤트 만들기

카드에 `"supply": true`를 달고, 선택지에 `"grant": "<아이템 id>"`를 넣으면 그 선택지를 고를 때 장비를 획득한다.
보급 이벤트는 한 번 등장하면 다시 나오지 않는다.

```json
{
  "id": "supply_head",
  "title": "보급 — SUMMERTECT 부스",
  "supply": true,
  "when": { "bands": ["오전"], "days": [2] },
  "choices": [
    { "label": "Flexer 캡을 받는다", "grant": "flexer_cap", "effects": {}, "result": "..." }
  ]
}
```

### 장비 수치 조정 (`data/items.json`)

각 장비는 `mods` 배열로 효과를 정의한다. 엔진은 `kind`만 해석하므로 숫자(`factor`/`delta`/`bands`)만 바꾸면 된다.

```json
"flexer_cap": {
  "name": "Flexer 캡", "slot": "머리",
  "spec": "UPF 자외선 차단 · 47g 경량 · packable",
  "desc": "정오·오후 체온 상승 -30%",
  "mods": [
    { "kind": "scale", "stat": "체온", "phase": "drift", "bands": ["정오","오후"], "direction": "up", "factor": 0.7 }
  ]
}
```

- `kind: "perTurn"` — 매 턴 고정 변화. `{ stat, delta }`
- `kind: "scale"` — 변화량에 배율. `{ stat, factor, phase, direction, bands? }`
  - `phase`: `"drift"`(시간대 패시브) · `"effect"`(선택 효과) · `"both"`
  - `direction`: `"up"`(증가분만) · `"down"`(감소분=소모만) · `"any"`
  - `bands`: 지정 시 해당 시간대에서만 적용
- `slot`: 머리 / 목 / 다리 / 하의. 같은 슬롯 장비는 나중 장착이 **교체**된다.
